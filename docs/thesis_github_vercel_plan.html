Fase 1 — Fundamentos (dias 1–2)
1
Crear el repositorio en GitHub
Nombre: thesis-cereal-climate. Empieza en privado. Luego lo abres cuando estes listo para publicar o compartir con tu director/a.
git init git remote add origin https://github.com/tu-usuario/thesis-cereal-climate
2
Configurar .gitignore para datos pesados
Los archivos .nc, .tif y las carpetas de datos crudos no van en git. Solo datos procesados pequenos y scripts.
*.nc *.tif *.hdf5 _temp_dask/ 01_Data/SST/ 01_Data/GDHY/ 01_Data/CEREALS/
3
Agregar LICENSE y CITATION.cff
CC BY 4.0 para documentos/resultados, MIT para codigo Python. CITATION.cff define como te citan — GitHub lo muestra con boton automatico.
Fase 2 — Migrar trabajo existente (dias 3–7)
4
Reorganizar scripts en src/paper1/ y src/paper2/
Mueve desde 02_Scripts/ y CORRELA_TODOS. Un commit por modulo con mensaje descriptivo.
git commit -m "feat(paper1): add SST-yield correlation module" git commit -m "feat(paper1): add SOM classification script"
5
Mover resultados a results/ y figuras a web/public/
Los GIFs (maize_yield, rgb_cereals) van a results/paper1/. Las figuras que quieres mostrar en el sitio van a web/public/figures/.
Fase 3 — Conectar Vercel (dia 8)
6
Crear sitio web basico en web/
Empieza simple: HTML + CSS estatico, o Astro si quieres algo mas estructurado. Una pagina de inicio + una seccion de resultados donde iras subiendo figuras.
7
Importar en Vercel y activar auto-deploy
vercel.com → New Project → Import from GitHub → selecciona web/ como root directory. Desde ese momento, cada push a main actualiza el sitio automaticamente.
Fase 4 — Citabilidad y DOI (dia 9)
8
Conectar Zenodo para DOI permanente
zenodo.org → conectar con GitHub → activar el repo → crear Release v0.1.0 en GitHub → Zenodo genera DOI automaticamente. Puedes poner el DOI en el README desde ese dia.
GitHub Release: v0.1.0 → Zenodo DOI: 10.5281/zenodo.XXXXXXX
Como proteger tu autoria siendo ciencia abierta al mismo tiempo
CC BY 4.0 — libre uso con atribucion obligatoria
Cualquiera puede usar, compartir y adaptar tu trabajo, pero siempre citandote. Tu nombre queda permanentemente asociado a los resultados. Es el estandar de ciencia abierta.
Git history = prueba legal de autoria
Cada commit tiene timestamp, autor y hash unico. Si alguien usa tu trabajo sin citarte, tienes evidencia irrefutable de que lo creaste primero. Es un registro publico con fechas.
Zenodo DOI — snapshot citable permanente
Cada GitHub Release genera un DOI que sobrevive aunque cierres el repo. Los papers pueden citarte con DOI antes de que publiques el articulo. Freeze de version en ese momento exacto.
CITATION.cff — instrucciones formales de citacion
Archivo que GitHub muestra con boton "Cite this repository". Cualquier gestor de referencias (Zotero, Mendeley) lo importa automaticamente con nombre, ano, titulo y DOI.
Empieza privado, abre cuando estes listo
Puedes dar acceso a tu director/a sin hacer el repo publico. Lo abres antes de enviar el paper a revista, o el dia de la defensa. El historial de commits ya esta protegiendo tu trabajo desde el primer dia.
